üìã Project Description

InsureMate is a comprehensive insurance data management system built on the NitroStack framework. It provides intelligent insurance plan recommendations, booking management, and user account handling through an AI-powered interface. The system leverages MongoDB for data storage, implements advanced caching mechanisms, and includes intelligent scoring algorithms to match users with the most suitable insurance plans.

The platform enables users to search, filter, and book insurance plans based on their profile, including age, salary, family size, and health conditions. It features a robust booking system with payment tracking and comprehensive status monitoring.

üîß Tools Description

  • 1. list_insurance

    Lists insurance records from MongoDB, filtered by salary, family members, and age. Supports pagination and custom MongoDB filters.

    Required: salary (annual salary for affordability filtering)
    Optional: familyMembers, age, limit, skip, filter
    Features: Affordability filtering (premium ‚â§ 10% of salary), family coverage matching, age range validation

  • 2. suggest_insurance_plan

    Provides personalized insurance plan suggestions using an intelligent scoring algorithm (0-100 match score). Returns top 5 plans with detailed reasons.

    Required: age, salary, familyMembers
    Optional: deficiencies (health conditions), insuranceType
    Scoring Factors: Age Match (30pts), Premium Affordability (25pts), Family Coverage (20pts), Pre-existing Conditions (25pts), Specific Conditions (15pts), Type Match (10pts)

  • 3. book_insurance_with_user

    Books an insurance plan by creating or updating user account and creating a booking record. Validates policy existence and handles payment information.

    Required: policyNumber, email, name, phoneNumber
    Optional: paymentMethod, startDate, years, transactionId, notes
    Features: User account management, booking creation, payment status tracking, coverage date calculation

  • 4. list_booking_status_by_email

    Retrieves all booking records for a user identified by email address. Returns complete booking history with full details sorted by most recent first.

    Required: email
    Returns: User information, all bookings with status, premium, coverage, payment details, dates, and notes

Example Usage Scenarios

Scenario 1: Finding Affordable Family Health Insurance

User Query: "I need health insurance for my family of 4. My salary is 500,000 per year and I'm 35 years old."

System Response: Uses list_insurance tool with salary=500000, familyMembers=4, age=35 to filter and return affordable plans.

Scenario 2: Getting Personalized Recommendations

User Query: "I have diabetes and need health insurance. I'm 40, earn 600,000, and have 3 family members."

System Response: Uses suggest_insurance_plan with deficiencies=["diabetes"] to get top 5 scored recommendations with detailed reasons.

Scenario 3: Booking an Insurance Plan

User Query: "I want to book policy POL-HDFC-HEA-0002"

System Response: Prompts for email, then uses book_insurance_with_user to create booking and user account.
